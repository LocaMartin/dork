  <strong>Dork</strong>
</p>

Dork is a Go-based command-line tool designed for for security researchers and penetration testers who need to automate Google dorking operations across multiple domains efficiently.

> I was bug hunting and encountered an issue. I wanted to perform dorking (advanced search operations to uncover information on the internet) but only for domains that were in scope, not the entire web. While there are many tools and scripts available online, none of them solved this particular issue. So I made a tool that solves this issue - "scoped dorking." This tool can search all domains for matching dorks but can narrow its radius and focus on domains in scope (domains.txt) or a single domain.

**Installation:**
```
go install github.com/LocaMartin/dork@latest
```

### usage:
single domain
```bash
dork -site examle.com -p 100 -t 10
dork-shell% inurl:"path" intext:"hello"
```
multi-domain
```bash
dork -site-list domains.txt -p 50 -t 50 -ds
dork-shell% inurl:"path" intext:"hello"
```
pass through other commands ( only domains allowed for dorking and proxychains allowed incase google blocks ip for heavey request like 1000 pages scan through pass )
```bash
cat domains.txt | sort -u | dork -p 100 -t 10 -o output.txt
# or 
proxychains | dork -site examle.com -p 100 -t 10
dork-shell% inurl:"path" intext:"hello"
```
if you have a large list of dorks you can sove it in a text file and `-dork-list` flag
```bash
dork -site-list domains.txt -p 50 -t 50 -dork-list dorks.txt
```

### flags:

```bash
-site example.com
-dork-list <dorks.txt>
-site-list <domains.txt>
-p: <10> for number of pages
-silent: did not print banner
-t: threads ( default 10 )
-o: output file ( output.txt )
-h: shows this help message on the terminal
-version: check version of the tool
```
`dorks.txt` file contents should be like:
```bash
inurl:path
intext:"hello sir"
intitle:"CVE-2025-XXXX"
site:nytimes.com cybersecurity
filetype:pdf confidential
intitle:"login page"
```
